Motivation

The Problem

Threat modeling is recognized by OWASP, NIST SP 800-154, and ISO/IEC 27034 as a foundational secure-development activity — yet adoption in mobile teams remains limited. Producing a model manually takes 2–5 days, requires specialist expertise, and existing tools provide no Android-specific analysis.

For Android applications, the attack surface is further compounded by the component model (activities, services, broadcast receivers, content providers) and the need to map every threat to specific OWASP MASVS v2 controls across 7 control families.

TMA bridges this gap by combining Androguard-based APK static analysis with LLM-driven context elicitation and structured STRIDE / LINDDUN generation — the only open-source tool to do all of this simultaneously.

Features

What It Does

🔍

APK Static Analysis

Extracts permissions, exported components, and API endpoints from APK binaries via Androguard — no source code required.

🧠

LLM Context Elicitation

Generates 6 targeted questions from the extracted context before synthesizing the threat model. Works with Anthropic, OpenAI, Ollama, or mock mode.

⚔️

STRIDE & LINDDUN

Supports both major threat modeling frameworks — STRIDE for security and LINDDUN for privacy — selectable per session.

🛡️

OWASP MASVS v2 Mapping

Every threat entry is automatically mapped to specific MASVS v2 controls across all 7 categories: STORAGE, CRYPTO, AUTH, NETWORK, PLATFORM, CODE, RESILIENCE.

🌐

Bilingual FR / EN

Full interface and generated output in French and English. Language preference persists across sessions via localStorage.

📦

Single-Command Deploy

Fully containerized with Docker Compose. The mock provider requires no API key — zero-config for exploration and CI.

Architecture

How It Works

TMA follows a three-tier client-server architecture. nginx routes /api/ requests to the FastAPI backend while serving the React SPA at the root path.

Frontend — React 18 + Vite · port 3000

Backend — FastAPI · Python 3.12 · port 8000

3-Step User Workflow

01 / UPLOAD

APK + Description

Optionally upload an APK (up to 100 MB) and provide a natural-language description of the application.

02 / QUESTIONS

Context Elicitation

Answer 6 LLM-generated questions. Select STRIDE or LINDDUN framework before launching generation.

03 / RESULTS

Threat Model

Threats tab · Risk Register · MASVS Mitigations. Export the full model as JSON.
